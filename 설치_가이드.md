# 📱 실시간 통역 앱 — 설치 & 모델 적용 완전 가이드

> 작성일: 2026-04-19

---

## 🗺️ 전체 흐름 한눈에 보기

```
[PC] Android Studio 설치
      ↓
[PC] 프로젝트 빌드 → APK 생성
      ↓
[PC→폰] APK 설치 (USB 또는 직접 전송)
      ↓
[PC] Gemma 모델 다운로드 (.task 파일, ~3-4GB)
      ↓
[PC→폰] ADB로 모델 파일 복사
      ↓
[폰] 앱 실행 → 모델 경로 설정 → 번역 시작!
```

---

## STEP 1. PC 환경 준비

### 1-1. Android Studio 설치
1. [https://developer.android.com/studio](https://developer.android.com/studio) 접속
2. **"Download Android Studio"** 클릭 → 설치 파일 실행
3. 설치 마법사 기본값으로 진행 (모두 Next)
4. 첫 실행 시 **SDK 자동 다운로드** 허용

> ✅ 설치 확인: Android Studio 실행 후 메인 화면 뜨면 OK

---

### 1-2. USB 디버깅 활성화 (핸드폰 설정)

**Galaxy 기준:**
```
설정 → 휴대전화 정보 → 소프트웨어 정보
→ "빌드번호" 7번 연속 탭 → 개발자 모드 활성화

설정 → 개발자 옵션 → USB 디버깅 ON
```

**Pixel 기준:**
```
설정 → 휴대전화 정보 → 빌드번호 7번 탭
설정 → 시스템 → 개발자 옵션 → USB 디버깅 ON
```

그 다음 **USB로 PC에 연결** → 폰에 "USB 디버깅 허용?" 팝업 → **허용**

---

## STEP 2. 앱 빌드 & 설치

### 2-1. 프로젝트 열기
1. Android Studio 실행
2. **File → Open** 클릭
3. `C:\Users\찬영\Desktop\안티그래비티 작업\RealtimeTranslator` 폴더 선택
4. **"Trust Project"** 클릭

### 2-2. Gradle 동기화
- 상단 알림바에 **"Sync Now"** 버튼 클릭  
  (또는 File → Sync Project with Gradle Files)
- 완료까지 2~5분 소요 (처음 실행 시 인터넷 다운로드)

### 2-3. 기기에 직접 설치 (가장 간단!)
1. 상단 바에서 연결된 폰 이름 선택 (예: `Samsung Galaxy S24 Ultra`)
2. **▶ Run** 버튼 클릭 (또는 `Shift + F10`)
3. 폰에 자동으로 앱 설치 & 실행

### 2-4. APK 파일로 설치 (USB 없이 전송할 때)
```
Build → Build Bundle(s)/APK(s) → Build APK(s)
완료 후 → "locate" 클릭
→ app/build/outputs/apk/debug/app-debug.apk 파일을 카카오톡/USB로 폰에 전송
→ 폰에서 파일 탭 → "출처를 알 수 없는 앱 허용" → 설치
```

## STEP 3. Gemma 모델 다운로드 (PC에서)

### 3-1. Kaggle에서 공식 모델 다운로드 (권장)

1. [https://www.kaggle.com/models/google/gemma](https://www.kaggle.com/models/google/gemma) 접속
2. **Gemma 4** → **Gemma4E4BIT** 선택
3. **Framework: "MediaPipe"** 선택 ← 중요!
4. `.task` 파일 다운로드 (약 **3~4GB**)

> ⚠️ MediaPipe 형식을 선택해야 합니다. PyTorch/JAX 형식은 앱에서 동작 안 합니다!

### 3-2. AI Edge Gallery 앱으로 모델 다운로드 (대안 - 가장 쉬움!)

구글이 만든 데모 앱을 활용하면 폰에서 직접 다운로드 가능:

1. Play Store에서 **"AI Edge Gallery"** 검색 & 설치
2. 앱 내에서 Gemma 모델 다운로드
3. 다운로드된 `.task` 파일 경로 확인 후 우리 앱에서 동일 경로 사용

---

## STEP 4. 모델 파일을 폰에 복사

### 방법 A: ADB 명령어 (가장 안정적)

PC에 ADB 설치 확인 (Android Studio 설치 시 자동 포함):
```powershell
# PowerShell에서 실행
$env:Path += ";$env:LocalAppData\Android\Sdk\platform-tools"
adb devices   # 폰이 연결되면 시리얼번호 출력됨
```

모델 복사:
```powershell
# 다운로드한 .task 파일을 폰으로 복사
adb push C:\Users\찬영\Downloads\gemma-4-e4b-it.task /sdcard/Download/

# 완료 확인
adb shell ls /sdcard/Download/gemma-4-e4b-it.task
```

### 방법 B: PC 파일 탐색기로 복사 (USB 연결 시)
1. USB 연결 → 폰에서 **"파일 전송"** 모드 선택
2. 파일 탐색기에서 폰 드라이브 열기
3. `내부 저장소 → Download` 폴더에 `.task` 파일 붙여넣기

---

## STEP 5. 앱에서 모델 경로 설정

1. **앱 실행** → 첫 화면에서 모델 경로 입력
2. 경로 입력:
   ```
   /storage/emulated/0/Download/gemma-4-e4b-it.task
   ```
3. **"모델 로드"** 버튼 탭
4. 로딩 완료 후 상단에 **녹색 "AI 준비됨"** 뱃지 확인

> ⏱️ 모델 로딩: 최초 1회 30초~1분 소요 (이후 빠름)

---

## STEP 6. 실제 사용하기

1. **언어 선택**: 상단 드롭다운 → 원문 언어 / 번역 언어 선택
   - 예: `한국어 → 영어` (회의에서 한국어로 말하면 영어로 번역)
2. **파란 마이크 🎤 버튼** 탭 → 말하기 시작
3. 화면에 **원문**(위)과 **번역**(아래) 실시간 표시
4. 빨간 **⏹ 버튼**으로 중지

## ❓ 자주 겪는 문제 & 해결법

| 문제 | 원인 | 해결법 |
|---|---|---|
| 앱 빌드 실패 | SDK 미설치 | Android Studio → SDK Manager → API 35 설치 |
| 폰이 ADB에 안 잡힘 | USB 디버깅 OFF | 폰 개발자 옵션 재확인 |
| 모델 로드 실패 | 경로 오류 | `/sdcard/` 대신 `/storage/emulated/0/` 사용 |
| 번역이 느림 | 기기 성능 부족 | E2B 모델로 교체 (8GB RAM) |
| 음성 인식 안 됨 | 마이크 권한 거부 | 설정 → 앱 → 실시간 통역 → 마이크 권한 허용 |
| OOM 크래시 | RAM 부족 | 다른 앱 종료, E2B 모델 사용 |
